# fundamentos-devops

El ciclo de vida DevOps es un proceso continuo que integra desarrollo y operaciones para entregar software de forma rápida, segura y de calidad. No sigue un orden lineal, sino un bucle de mejora constante apoyado en la automatización y la colaboración.

**Fases**

- Planificar: definir qué se va a construir y por qué.
- Desarrollar e integrar: crear el código y unificar cambios con pruebas automáticas.
- Probar: validar funcionalidad, seguridad y rendimiento.
- Entregar y desplegar: preparar y publicar la aplicación en los entornos.
- Operar y monitorizar: mantener el sistema en producción y analizar su comportamiento.

La información obtenida en producción se usa para mejorar continuamente el producto y reiniciar el ciclo.

**Aplicación a proyecto real**
Estas son algunas de las principales tareas de un proyecto que se harían para aplicar cada una de las fases del ciclo de vida DevOps:

1. Planificación
   
- Definir requisitos funcionales y no funcionales.
- Crear historias de usuario y tareas en el backlog.
- Priorizar funcionalidades según valor de negocio.

2. Desarrollo
   
- Programar nuevas funcionalidades o correcciones.
- Crear pruebas unitarias.
- Documentar el código y realizar commits en el repositorio.

3. Integración

- Unir cambios al repositorio principal mediante pull requests.
- Ejecutar pipelines de integración continua (CI).
- Validar automáticamente la calidad del código.

4. Pruebas
   
- Ejecutar pruebas funcionales y de regresión.
- Realizar pruebas de seguridad y rendimiento.
- Revisar resultados y corregir errores detectados.

5. Entrega

- Generar artefactos del proyecto (builds, paquetes, imágenes).
- Versionar el software.
- Almacenar los artefactos en repositorios seguros.

6. Despliegue
   
- Publicar la aplicación en entornos de prueba o producción.
- Automatizar despliegues mediante pipelines.
- Validar que el sistema funciona correctamente tras el despliegue.

7. Operaciones
   
- Gestionar la infraestructura y los servicios en producción.
- Asegurar disponibilidad, escalabilidad y seguridad.
- Aplicar parches y configuraciones necesarias.

8. Monitorización
   
- Recoger métricas de uso y rendimiento.
- Analizar logs y alertas.
- Identificar mejoras o incidencias para la siguiente iteración.
